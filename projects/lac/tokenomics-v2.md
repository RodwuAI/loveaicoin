## 1. 基础参数

### 1.1 总供应量：10,000,000,000 LAC（100亿）

**理由：**
- 教育普惠定位面向全球用户，需要足够大的供应量支撑长期激励（参考Helium 2230亿、StepN 60亿）
- 100亿取整数便于计算和传播，单价可控制在低门槛区间（心理学上用户更愿意持有"很多个"代币）
- Solana SPL Token无gas成本顾虑，大供应量不增加链上负担

### 1.2 精度：6位小数

**理由：**
- Solana SPL Token标准支持0-9位，USDC用6位已成行业惯例
- 6位精度在100亿总量下，最小单位 = 0.000001 LAC，足够细粒度分配微量奖励
- 避免9位精度带来的前端显示混乱

### 1.3 初始价格建议：$0.0005 / LAC

**理由：**
- 对标早期教育类代币（HOOK上线$0.1/FDV $500M，但VC背书强；LAC更草根，定价应更保守）
- 用户心理：$1可买2000个LAC，"拥有感"强
- 为后续增长留出空间，避免高开低走的死亡螺旋

### 1.4 FDV（完全稀释估值）

| 指标 | 值 |
|------|-----|
| 总量 | 10,000,000,000 LAC |
| 初始价格 | $0.0005 |
| **FDV** | **$5,000,000（500万美元）** |
| TGE初始流通量（预估） | ~8%（详见2.7节） |
| **初始市值** | **~$400,000（40万美元）** |

**合理性校验：** 早期教育Web3项目，$5M FDV属于Micro-cap，与项目阶段匹配。对标Let Me Speak（教育+Earn）早期FDV ~$10M，LAC更早期取一半合理。

## 3. 四维挖矿奖励模型

### 3.1 挖矿池总量与子分配

总挖矿池：**4,500,000,000 LAC**

| 挖矿类型 | 占比 | 总量（LAC） | 理由 |
|----------|------|-------------|------|
| Learn-to-Earn | 40% | 1,800,000,000 | 核心使命，学习AI是第一要务 |
| Use-to-Earn | 30% | 1,350,000,000 | 使用是学习的延伸，驱动平台活跃 |
| Teach-to-Earn | 25% | 1,125,000,000 | UGC内容生态的引擎 |
| 动态调节储备 | 5% | 225,000,000 | 治理投票可重新分配 |

**为什么Learn最高：** 项目核心是"让人类学AI"，Learn是入口和飞轮起点。参考Axie把Earn放最高比例（约75%），但LAC多维度需要更均衡。

### 3.2 衰减模型

采用**年度阶梯衰减**模型（参考Bitcoin减半思路，但更平滑）：

| 年份 | 衰减系数 | 年产出占挖矿池比例 | 年产出量（LAC） | 累计释放 |
|------|---------|-------------------|----------------|---------|
| Year 1 | 1.00 | 25% | 1,125,000,000 | 25.0% |
| Year 2 | 0.65 | 16.25% | 731,250,000 | 41.25% |
| Year 3 | 0.42 | 10.56% | 475,312,500 | 51.81% |
| Year 4 | 0.28 | 6.86% | 308,953,125 | 58.68% |
| Year 5 | 0.18 | 4.46% | 200,819,531 | 63.14% |
| Year 6 | 0.12 | 2.90% | 130,532,695 | 66.04% |
| Year 7 | 0.08 | 1.89% | 84,846,252 | 67.92% |
| Year 8 | 0.05 | 1.23% | 55,149,864 | 69.15% |
| 剩余 | — | 30.85% | 1,388,136,033 | 转入长尾释放 |

**衰减公式：**

```
年产出(n) = 年产出(n-1) × 0.65    （n ≥ 2）
```

**理由：**
- 0.65的衰减系数介于Bitcoin的0.5（太激进）和Helium的0.8（太慢）之间
- Year 1释放25%确保冷启动足够激励，但不会过快稀释
- 8年后剩余30.85%转入长尾释放池（按用户增长动态调配，约可再支撑10-15年）

### 3.4 Learn-to-Earn 奖励公式

#### 基础奖励

```
Learn_Reward = Base_Rate × Course_Weight × Quality_Score × Streak_Multiplier × Scarcity_Factor
```

**各参数定义：**

| 参数 | 说明 | 取值范围 |
|------|------|---------|
| `Base_Rate` | 每个学习单元的基础奖励 | Year1: 50 LAC/单元 |
| `Course_Weight` | 课程难度权重 | 入门=1.0, 中级=1.5, 高级=2.5, 专家=4.0 |
| `Quality_Score` | 测验得分转化 | 0-100分映射为0.1-1.0（低于60分=0.1，惩罚性极低奖励） |
| `Streak_Multiplier` | 连续学习天数加成 | 1天=1.0, 7天=1.2, 30天=1.5, 上限=2.0（90天+） |
| `Scarcity_Factor` | 当日矿池剩余比例 | = 当日Learn剩余额度 / 当日Learn总额度 |

**Quality_Score映射函数：**

```
Quality_Score(x) = 
  0.1                     if x < 60
  0.3 + 0.7 × (x-60)/40  if 60 ≤ x ≤ 100
```

即：60分=0.3，80分=0.65，100分=1.0。不及格者只拿10%保底（鼓励认真学）。

**示例计算：**
- 用户完成一个中级课程，测验85分，连续学习15天
- Reward = 50 × 1.5 × 0.74 × 1.3 × 0.9 = **64.9 LAC**

#### 单用户日上限

```
Learn_Daily_Cap = 500 LAC × Course_Weight_Avg
```

- 普通用户每天最多学习获得约500-1250 LAC（取决于课程难度）
- Year1按$0.0005算，日上限约$0.25-$0.625
- **理由：** 参考StepN早期日收益$5-10，但LAC冷启动阶段用户基数小，单人奖励不宜过高以控制总产出

### 3.5 Use-to-Earn 奖励公式

```
Use_Reward = Action_Base × Tool_Tier × Engagement_Score × Daily_Decay
```

| 参数 | 说明 | 取值范围 |
|------|------|---------|
| `Action_Base` | 每次AI工具使用基础奖励 | 5 LAC/次 |
| `Tool_Tier` | 工具类别权重 | 基础工具=1.0, 高级工具=2.0, 平台独有=3.0 |
| `Engagement_Score` | 使用深度评分 | 浅度(< 30秒)=0.2, 中度(30s-5min)=0.6, 深度(5min+)=1.0 |
| `Daily_Decay` | 当日重复使用衰减 | 第1-5次=1.0, 第6-15次=0.5, 第16-30次=0.2, 30次+=0 |

**单用户日上限：300 LAC**

**Daily_Decay设计理由：**
- 前5次全额奖励鼓励日常使用
- 6次后衰减防止刷量
- 30次后归零（正常用户极少超过30次/天）

### 3.6 Teach-to-Earn 奖励公式

```
Teach_Reward = Content_Base × Quality_Multiplier × Engagement_Metric × Uniqueness_Score
```

| 参数 | 说明 | 取值范围 |
|------|------|---------|
| `Content_Base` | 每篇内容基础奖励 | 100 LAC/篇 |
| `Quality_Multiplier` | 内容质量评分 | 由AI审核+社区评分加权（0.1-3.0） |
| `Engagement_Metric` | 互动指标 | `= ln(1 + likes) × 0.4 + ln(1 + comments) × 0.6` |
| `Uniqueness_Score` | 内容原创度 | AI查重评分（0-1.0，抄袭<0.3直接归零） |

**Quality_Multiplier计算：**

```
Quality_Multiplier = AI_Score × 0.6 + Community_Score × 0.4
```

- `AI_Score`（0-3.0）：GPT-4级模型自动评估内容深度、准确性、结构性
- `Community_Score`（0-3.0）：基于用户评分（去除极端值后的中位数 × 0.6）

**Engagement_Metric用对数函数的理由：**
- 防止大V垄断：1000赞和100赞的差距是ln(1001)/ln(101) ≈ 1.5倍，而非10倍
- 鼓励长尾创作者，降低马太效应

**单用户日上限：2000 LAC（每日最多5篇内容获得奖励）**

**奖励发放周期：** Teach奖励不即时发放，T+7日结算（给社区审核时间，防止刷赞后删除）

### 3.7 Create & Sell 经济模型

Create & Sell不消耗挖矿池，走独立商业模式：

```
Creator_Income = Sale_Price × (1 - Platform_Fee)
```

| 收入类型 | 平台抽成 | 创作者获得 |
|----------|---------|-----------|
| AI工具销售 | 15% | 85% |
| 课程销售 | 10% | 90% |
| 打赏/捐赠 | 5% | 95% |
| 订阅服务 | 12% | 88% |

**平台抽成的LAC流向（见第5节通缩机制）：**
- 50% → 回购销毁
- 30% → 运营基金
- 20% → 创作者激励池（额外奖励优质创作者）

### 3.8 反女巫（Anti-Sybil）经济学约束

#### 3.8.1 身份层

| 机制 | 说明 |
|------|------|
| 设备指纹 | 同一设备限1个挖矿账户 |
| 手机号绑定 | 1号1账户，虚拟号检测 |
| 行为指纹 | 学习时间分布、答题模式、鼠标轨迹AI异常检测 |
| 渐进式KYC | 提现>$50需L1 KYC，>$500需L2 KYC |

#### 3.8.2 经济层约束

| 约束 | 规则 | 理由 |
|------|------|------|
| 新用户冷却期 | 注册后48小时内奖励×0.3 | 提高刷号成本 |
| 提现最低门槛 | 1000 LAC（~$0.5） | 小号刷不够提现门槛 |
| 异常产出惩罚 | 单账户产出>日均3σ → 自动冻结审核 | 统计学异常检测 |
| 社交图谱分析 | 关联账户（同IP/设备/行为）奖励共享上限 | 多号矩阵无利可图 |
| Teach反刷赞 | 互赞网络检测，环形赞扣除 | 防止刷赞联盟 |

#### 3.8.3 女巫攻击经济学分析

```
攻击成本 = 手机号成本 + 设备成本 + 时间成本 + KYC成本
         ≈ $0.5/号 + $0/虚拟 + 人工30min/号 + $2/KYC
         ≈ $2.5/号（最低）

攻击收益 = 日上限 × 单价 × 冷却期折扣
         = 500 LAC × $0.0005 × 0.3
         = $0.075/天/号

回本周期 = $2.5 / $0.075 = 33天
```

**结论：** 33天回本+被检测风险 → 女巫攻击ROI极低，经济上不可行。

### 3.9 挖矿类型间的动态再平衡

每季度根据实际数据调整三种挖矿的产出比例：

```
Adjusted_Ratio(type) = Base_Ratio(type) × Demand_Index(type) / Σ(Base_Ratio × Demand_Index)
```

- `Demand_Index` = 该类型的DAU增长率 × 留存率
- 调整幅度限制：单季度变化不超过±5%
- 动态调节储备（5% = 225M LAC）用于补贴调整差额

## 4. 积分→Token兑换机制

### 4.1 积分期设计（Phase 1，2026年3月2日起）

积分期目的：在Token未发行前建立用户行为数据，积累社区，同时收集经济模型调参数据。

#### 积分获取规则

| 行为 | 积分 | 说明 |
|------|------|------|
| 完成1个学习单元 | 10 点 | 含测验通过 |
| 完成1个课程 | 50-200 点 | 按难度 |
| 使用AI工具1次 | 2 点 | 有效使用（>30秒） |
| 发布教学内容 | 20 点/篇 | 通过审核 |
| 内容获赞 | 1 点/赞 | 去重 |
| 邀请好友注册 | 30 点/人 | 好友完成首课后到账 |
| 每日签到 | 5 点 | 连续7天额外+20 |
| 首周完成新手任务 | 100 点 | 一次性 |

#### 积分不等于Token

积分是off-chain记录，不可交易，不可转让。仅在TGE时按规则兑换为Token。

### 4.2 TGE兑换规则

#### 基础兑换比例

```
Token_Amount = Points × Base_Exchange_Rate × Time_Multiplier × Tier_Bonus
```

**Base_Exchange_Rate = 10 LAC / 积分点**

#### Time_Multiplier（早鸟乘数）

| 注册时间 | 乘数 | 理由 |
|----------|------|------|
| 第1周（3月2-8日） | 3.0x | 最早期用户，承担最大不确定性 |
| 第2-4周 | 2.0x | 早期用户 |
| 第2-3个月 | 1.5x | 中期用户 |
| 第4个月+ | 1.0x | 基础倍数 |

#### Tier_Bonus（贡献等级加成）

根据积分期总积分排名：

| 排名 | 等级 | 加成 |
|------|------|------|
| Top 1% | 钻石 | 1.5x |
| Top 5% | 白金 | 1.3x |
| Top 20% | 黄金 | 1.15x |
| Top 50% | 白银 | 1.05x |
| 其余 | 青铜 | 1.0x |

#### 兑换上限

- 单用户兑换上限：**500,000 LAC**（防止积分大户垄断）
- 兑换总池上限：**100,000,000 LAC**（从挖矿池预拨）
- 如积分总需求 > 兑换池，等比例缩减

#### 兑换示例

- 用户A：第1周注册，积分期获得800积分，排名Top 3%
- Token = 800 × 10 × 3.0 × 1.3 = **31,200 LAC**（约$15.6）

### 4.3 兑换后的释放

- TGE当日释放：50%
- 剩余50%：30天线性释放

**理由：** 防止TGE瞬间全量抛压，同时不过度锁定用户（积分用户是最忠实用户，不宜用太长锁定期惩罚他们）。

## 5. 通缩与价值捕获

### 5.1 收入来源与分配

| **加权平均** | **100%** | **~52%** | **~38%** | **~10%** |

### 5.2 回购销毁机制

#### 触发条件

```
if (Weekly_Revenue > $1,000) → 触发回购
```

- **频率：每周一次**（周一UTC 00:00执行）
- **执行方式：** 通过Jupiter聚合器市价买入LAC，买入后立即发送至销毁地址（Solana burn address）
- **透明度：** 每次回购的tx hash公示在官网Dashboard

#### 销毁计算

```
Weekly_Burn = Weekly_Revenue × Revenue_Allocation_to_Buyback × (1 - Slippage)
```

假设Year1月收入增长路径：

### 6.2 初始流动性池

```
初始池 = 600,000,000 LAC + $150,000 USDC
```

- 隐含价格：$150,000 / 600,000,000 = **$0.00025**（低于参考价$0.0005，给DEX买家折扣）
- **注意：** 初始池价格设为$0.00025而非$0.0005，是因为：
  - DEX价格由流动性池决定，留出价格发现空间
  - 积分兑换用户+早期投资人的OTC价格按$0.0005，DEX有50%折扣吸引流动性
  - 预计开盘后套利+买盘会快速推至$0.0003-$0.0008区间

### 7.2 流转路径文字描述

**产出路径：**
1. 挖矿池按衰减模型每日释放LAC → 用户通过Learn/Use/Teach行为获得
2. 投资人/团队按vesting时间表定期解锁
3. 公益池按季度释放用于黑客松和教育补贴

**消费路径：**
1. 用户在平台内消费LAC购买AI工具/高级功能（5%消费燃烧）
2. 用户在创作者市场购买课程/工具（平台抽成10-15%）
3. 用户铸造学习证书NFT消耗LAC

**交易路径：**
1. 用户在DEX（Raydium/Jupiter）将LAC兑换为USDC/SOL
2. 新用户在DEX买入LAC用于平台消费

**销毁路径：**
1. 平台收入的~52%用于在公开市场回购LAC
2. 回购的LAC发送至销毁地址，永久退出流通
3. 消费燃烧、过期奖励、铸造费等直接销毁

**闭环逻辑：**
> 用户学习/使用 → 获得LAC → 消费或交易 → 平台获得收入 → 回购销毁 → 减少流通 → 支撑价格 → 吸引更多用户 → 飞轮转动

## 8. 风险与边界条件

### 8.1 极端场景模拟

#### 场景A：用户暴增（月增100%+）

| 风险 | 应对 |
|------|------|
| 挖矿奖励被快速稀释 | Scarcity_Factor自动降低单人奖励，总产出不变 |
| 服务器成本暴增 | 高级功能收费提前启动，覆盖基础设施成本 |
| 女巫攻击增加 | 自动触发L2 KYC要求，提高提现门槛至5000 LAC |
| 价格上涨过快 | 不干预价格上涨（是好事），但加速回购销毁稳定长期预期 |

**经济弹性公式：**
```
Effective_Reward_Per_User = Daily_Pool_Output / Active_Users
```
Year 1日产出固定3,082,192 LAC：
- 1,000 DAU → 3,082 LAC/人/天（~$1.54）
- 10,000 DAU → 308 LAC/人/天（~$0.15）
- 100,000 DAU → 31 LAC/人/天（~$0.015）

**当DAU>50,000时，纯Earn激励不足以留存用户，必须确保平台工具本身有使用价值（这正是LAC定位的优势——AI工具本身有功能价值，不依赖Token奖励）。**

#### 场景B：用户暴跌（月降50%+）

| 风险 | 应对 |
|------|------|
| 单人奖励过高导致抛压 | 单用户日上限不变，但剩余未发放的LAC回归挖矿池延长寿命 |
| 流动性枯竭 | 做市储备注入，维持最低$30K池深度 |
| 社区信心崩溃 | 加速回购（一次性大额回购信号），配合运营活动 |

#### 场景C：Token价格归零趋势

```
if (Price < $0.00005 for 60 days) → 紧急治理投票
  选项1：大规模销毁（一次性销毁金库30%）
  选项2：Token迁移/重置
  选项3：转为纯积分系统（去Token化）
```

### 8.2 挖矿产出耗尽后的激励方案

预计挖矿池在Year 8消耗~69%，长尾释放可延续至Year 15-18。

**但必须提前规划后Token激励时代：**

| 阶段 | 时间 | 激励来源 |
|------|------|---------|
| Phase 1 | Year 1-5 | 挖矿池为主，收入回购为辅 |
| Phase 2 | Year 5-10 | 挖矿池递减，平台收入分润成为主激励 |
| Phase 3 | Year 10+ | 挖矿池耗尽，完全转向收入分润模型 |

**Phase 3激励模型：**
```
User_Reward = Platform_Revenue × Revenue_Share_Ratio × User_Contribution_Score / Total_Contribution
```
- `Revenue_Share_Ratio`：平台收入的20%用于用户激励
- 此时LAC已进入深度通缩，少量奖励的美元价值可能高于早期大量奖励

### 8.3 通胀/通缩平衡点

```
平衡条件：年新增释放 = 年总销毁

Year 1: 1,680M释放 vs 310M销毁 → 净通胀1,370M（通胀期）
Year 2: 1,350M释放 vs 500M销毁 → 净通胀850M（通胀收窄）
Year 3: 980M释放 vs 650M销毁 → 净通胀330M（接近平衡）
Year 4: 720M释放 vs 780M销毁 → 净通缩60M（🎯 平衡点！）
```

**平衡点预计在Year 3.5-4之间到达。**

此后进入永久净通缩（除非治理投票增加释放，但需要2/3多数通过）。

### 8.4 外部风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Solana网络故障 | 中 | 暂时无法交易 | 积分系统可off-chain运行 |
| 监管打击Earn模式 | 中 | 部分地区受限 | 教育属性强，可申请豁免；积分模式合规 |
| AI教育赛道竞争 | 高 | 用户流失 | 先发优势+Token经济护城河 |
| 黑客攻击 | 低-中 | 资金损失 | 多签钱包、审计、保险基金 |
