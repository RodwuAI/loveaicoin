## 一、Token基础设计

### 1.1 基本参数

| 参数 | 值 |
|------|-----|
| Token名称 | LAC (Love AI Coin) |
| 链 | Solana (SPL Token) |
| 总供应量 | **1,000,000,000 (10亿)** |

- 10亿是Solana生态主流量级（BONK 100万亿太大，JTO 10亿合理）

## 二、三维挖矿经济模型

### 2.1 挖矿池分配

总挖矿池：400,000,000 LAC

| 维度 | 比例 | 数量 | 理由 |
|------|------|------|------|
| 🤗 拥抱挖矿 (Hug-to-Earn) | 45% | 180,000,000 | 社区是核心，社交传播驱动增长 |
| 📚 学习挖矿 (Learn-to-Earn) | 25% | 100,000,000 | 知识壁垒带来用户粘性 |
| 🔧 实用挖矿 (Use-to-Earn) | 30% | 120,000,000 | 产品使用创造真实价值 |

### 2.2 全局衰减模型

采用**半衰期衰减**，参考BTC减半逻辑但更平滑：

```
每月释放量 = 基础月释放量 × (0.5)^(月份/18)
```

半衰期 = 18个月。即每18个月，月释放量减半。

| 时段 | 月释放量(近似) | 该时段累计释放 |
|------|----------------|----------------|
| 第1-6月 | ~11.5M/月 | ~69M |
| 第7-12月 | ~9.0M/月 | ~54M |
| 第13-18月 | ~7.0M/月 | ~42M |
| 第19-24月 | ~5.5M/月 | ~33M |
| 第25-36月 | ~3.5M/月 | ~42M |
| 第37-60月 | ~1.5M/月 | ~36M |
| **合计** | | **~276M** |

剩余~124M作为长尾储备，在第5年后按治理投票决定用途。

### 2.3 🤗 拥抱挖矿 (Hug-to-Earn) 详细设计

**核心理念：** 社交互动=价值创造。传播爱、帮助他人、建设社区都能挖矿。

#### 行为类型与奖励

| 行为 | 基础奖励(LAC) | 每日上限 | 说明 |
|------|---------------|----------|------|
| 每日签到 | 5 | 1次 | 连续签到有加成 |
| 邀请新用户(注册+完成首任务) | 50 | 5人 | 被邀请人必须完成首个任务 |
| 社群互动(有效发言) | 2 | 20次 | AI判定有意义的发言 |
| 帮助他人(被标记有用) | 15 | 10次 | 社区成员点赞/标记 |
| 内容创作(推文/视频) | 30-200 | 3次 | 根据质量AI评分 |
| 参与治理投票 | 10 | 每提案1次 | 鼓励去中心化治理 |
| 情感互动(与小嘀嗒对话) | 3 | 10次 | 深度对话质量加成 |

#### 连续签到加成

```
签到奖励 = 5 × min(连续天数 × 0.1 + 1, 3.0)
```

| 连续天数 | 倍率 | 实际奖励 |
|----------|------|----------|
| 1天 | 1.1x | 5.5 |
| 7天 | 1.7x | 8.5 |
| 14天 | 2.4x | 12.0 |
| 20天+ | 3.0x(封顶) | 15.0 |

#### 内容创作评分公式

```
内容奖励 = 基础分(30) × 质量系数(0.5-3.0) × 传播系数
质量系数 = AI评分(原创性40% + 深度30% + 相关性30%)
传播系数 = min(1 + ln(有效互动数+1)/5, 2.0)
```

#### 防作弊机制

| 风险 | 对策 |
|------|------|
| 机器人刷签到 | 集成验证码 + 设备指纹 + 行为分析 |
| 假邀请（自己注册小号）| 被邀请人需完成3个不同任务 + IP/设备去重 |
| 低质量刷发言 | AI语义分析，重复/无意义内容不计分 |
| 互刷点赞 | 社交图谱分析，环形互赞检测降权 |
| 抄袭内容 | AI查重，相似度>70%不计分 |

### 2.4 📚 学习挖矿 (Learn-to-Earn) 详细设计

**核心理念：** 学习AI和Web3知识，提升认知=挖矿。参考Galxe+RabbitHole模式。

#### 课程体系与奖励

| 课程等级 | 内容示例 | 完成奖励(LAC) | 时长 |
|----------|----------|---------------|------|
| L1 入门 | 什么是区块链/AI基础 | 20 | 5-10分钟 |
| L2 进阶 | Solana生态/提示词工程 | 50 | 15-30分钟 |
| L3 高级 | DeFi策略/AI Agent开发 | 150 | 30-60分钟 |
| L4 专家 | 独立项目/论文解读 | 500 | 2-5小时 |
| 每日快问(3题) | 随机AI/Web3知识 | 8 | 2分钟 |

#### 学习积分公式

```
学习奖励 = 课程基础奖励 × 正确率系数 × 首次加成
正确率系数 = (正确题数/总题数)^2    // 鼓励真正学会
首次加成 = 首次完成1.5x, 重复完成0.3x
```

**每日快问细节：**
- 从题库随机抽取3题，答对1题得2LAC，全对得8LAC（含bonus 2LAC）
- 题库由社区贡献+AI生成，每周更新50题

#### 防作弊机制

| 风险 | 对策 |
|------|------|
| 分享答案 | 题目动态生成，每人不同变体 |
| 速刷课程不学 | 设最短学习时间，随机插入理解题 |
| 多账号刷课程 | 同设备/IP限制，KYC可选但有额外加成 |
| AI代答 | 限时作答 + 行为时间戳分析 |

### 2.5 🔧 实用挖矿 (Use-to-Earn) 详细设计

**核心理念：** 使用LAC生态产品产生真实价值=挖矿。这是经济模型可持续性的关键。

#### 实用行为与奖励

| 行为 | 奖励(LAC) | 每日上限 | 说明 |
|------|-----------|----------|------|
| 使用AI Agent完成任务 | 5-25 | 20次 | 按任务复杂度 |
| 贡献训练数据（标注/反馈）| 10 | 30次 | 帮助AI进化 |
| 使用LAC支付(消费) | 消费额2%返还 | 500LAC | 生态内消费激励 |
| API调用(开发者) | 1/100次调用 | 100LAC | 鼓励开发者集成 |
| Bug报告(有效) | 100-1000 | 不限 | 按严重程度 |
| 数据贡献(匿名化) | 15 | 5次 | 隐私保护前提下 |

#### 实用价值评分

```
实用奖励 = 基础奖励 × 价值系数 × 频率衰减
价值系数 = min(任务复杂度评分/5, 3.0)
频率衰减 = 1/(1 + 0.05 × 当日同类行为次数)  // 防止单一行为刷量
```

#### 防作弊机制

| 风险 | 对策 |
|------|------|
| 虚假API调用 | 调用必须产生有效结果，空调用不计 |
| 低质量数据标注 | 交叉验证，与其他用户标注一致性>80%才计分 |
| 自买自卖刷消费 | 交易图谱分析，闭环交易降权 |

### 2.6 综合激励：爱值系统 (Love Score)

三个维度不是孤立的。引入**爱值(Love Score)**作为综合加成系数：

```
爱值 = 0.45 × 拥抱分 + 0.25 × 学习分 + 0.30 × 实用分
各维度分 = min(该维度累计行为评分 / 该维度满分, 100)
```

**爱值等级与加成：**

| 爱值范围 | 等级 | 全局挖矿加成 |
|----------|------|-------------|
| 0-20 | 🌱 种子 | 1.0x |
| 21-40 | 🌿 发芽 | 1.1x |
| 41-60 | 🌸 开花 | 1.25x |
| 61-80 | 🌳 成长 | 1.4x |
| 81-100 | 💎 觉醒 | 1.6x |

**设计意图：** 鼓励用户均衡参与三个维度，而非只刷单一维度。纯社交党或纯学习党无法达到最高等级。

## 三、通缩与价值捕获

### 3.1 燃烧机制

| 燃烧来源 | 燃烧比例 | 预估月燃烧量 |
|----------|----------|-------------|
| AI Agent使用费 | 收入的30%用于买入燃烧 | 视采用率 |
| NFT交易版税 | 版税的50%燃烧 | 视交易量 |
| 高级功能解锁费 | 100%燃烧 | ~200K LAC/月(成熟期) |
| 惩罚燃烧(作弊者) | 没收的100%燃烧 | ~50K LAC/月 |
| 爱值重置费 | 100%燃烧 | ~30K LAC/月 |

**目标：** 成熟期（12个月后）月燃烧量 > 月挖矿释放量的20%，逐步趋向通缩。

### 3.2 回购机制

- **收入来源：** AI Agent订阅费、企业API收入、合作伙伴费用
- **回购规则：** 每季度将协议收入的25%从公开市场回购LAC
- **回购后处理：** 50%燃烧 + 50%注入国库
- **透明度：** 回购交易哈希链上公开，季度回购报告

### 3.3 质押收益模型

#### 单币质押

| 锁仓期 | 基础APY | 来源 |
|--------|---------|------|
| 灵活(随取) | 5% | 国库拨款 |
| 30天 | 12% | 国库拨款+协议收入 |
| 90天 | 20% | 国库拨款+协议收入 |
| 180天 | 30% | 国库拨款+协议收入+挖矿加速 |

**APY调节机制：**
```
实际APY = 基础APY × (目标质押率/实际质押率)
目标质押率 = 40%流通量
```
- 质押率低于40% → APY上升 → 吸引质押
- 质押率高于40% → APY下降 → 释放流动性

#### 质押额外权益

| 质押量 | 权益 |
|--------|------|
| ≥1,000 LAC | 投票权 |
| ≥10,000 LAC | 提案权 + 挖矿1.1x加成 |
| ≥100,000 LAC | 高级AI功能 + 挖矿1.2x加成 |
| ≥1,000,000 LAC | 节点资格 + 收入分红 |

### 3.4 LP激励

**目标池：** LAC/SOL (Raydium) + LAC/USDC (Orca)

| 阶段 | 月LP激励预算 | 来源 |
|------|-------------|------|
| 第1-3月 | 3,000,000 LAC | 生态发展池 |
| 第4-6月 | 2,000,000 LAC | 生态发展池 |
| 第7-12月 | 1,200,000 LAC | 协议收入+生态池 |

## 四、竞品经济模型对比

### 4.1 对标分析

| 维度 | STEPN (GMT/GST) | Axie (AXS/SLP) | Galxe (GAL) | **LAC** |
|------|-----------------|-----------------|-------------|---------|
| 模式 | Move-to-Earn | Play-to-Earn | Learn-to-Earn | **三维融合X-to-Earn** |
| 双Token | ✅ GMT+GST | ✅ AXS+SLP | ❌ 单Token | **❌ 单Token** |
| 入场门槛 | 高(需买鞋NFT,$100-1000) | 极高(需买3只Axie,$500+) | 低(免费) | **低(免费)** |
| 每日收入上限 | ~5-50 GST | ~75-150 SLP | 任务制 | **多维度叠加** |
| 死亡螺旋 | ⚠️ 已发生(GST暴跌99%) | ⚠️ 已发生(SLP暴跌99%) | ✅ 无(非挖矿模式) | **多维对冲+燃烧** |
| 价值来源 | 运动数据(弱) | 游戏内消耗(弱) | 品牌合作(中) | **AI服务+数据+社区(强)** |
| 通缩设计 | 升级燃烧 | 繁殖消耗 | 有限 | **多层燃烧+回购** |

### 4.2 STEPN教训与LAC的应对

| STEPN问题 | 原因 | LAC解决方案 |
|-----------|------|------------|
| GST价格崩盘 | 产出无限+需求靠新用户 | 半衰期衰减+真实使用场景消耗 |
| 高NFT门槛形成庞氏 | 新玩家买鞋的钱付老玩家收益 | 零门槛进入，收益来自生态价值 |
| 单维度枯燥 | 只有走路 | 三维度交叉激励，玩法丰富 |
| 代币无内在价值 | GST只能升级鞋子 | LAC可支付AI服务、治理、质押分红 |

### 4.3 LAC经济模型核心优势

1. **零入场门槛：** 不需要买NFT/鞋子，注册即可挖矿，降低庞氏风险
2. **三维对冲：** 单一维度衰退不会导致整体崩盘，用户可在维度间迁移
3. **真实价值锚定：** AI Agent服务是真实需求，不是空气叙事
4. **单Token简洁设计：** 避免双Token的复杂性和SLP式通胀崩盘
5. **AI驱动的防作弊：** 利用自身AI能力检测作弊，这是Web3项目的独特优势

## 五、风险分析

### 5.1 死亡螺旋风险

**场景：** 币价下跌 → 挖矿收益下降 → 用户流失 → 卖压增加 → 币价继续跌

**对策矩阵：**

| 防线 | 机制 | 触发条件 |
|------|------|----------|
| 第一道 | 自动增加质押APY | 质押率<30% |
| 第二道 | 加速回购燃烧(从25%提至50%) | 币价连续7天低于30日均价20% |
| 第三道 | 减缓挖矿释放速度(50%) | 流通量增速>币价跌幅 |
| 第四道 | 国库紧急买盘 | 极端情况，治理投票通过 |
| 根本 | 产品本身有使用价值 | 确保AI服务质量持续提升 |

### 5.2 通胀压力控制

**关键指标监控：**

```
通胀率 = (月新释放量 - 月燃烧量) / 月初流通量 × 100%
```

| 净通胀率 | 状态 | 行动 |
|----------|------|------|
| <2%/月 | 🟢 健康 | 正常运营 |
| 2-5%/月 | 🟡 警戒 | 增加燃烧场景 |
| 5-10%/月 | 🟠 危险 | 减缓释放+加速回购 |
| >10%/月 | 🔴 紧急 | 暂停部分挖矿+紧急治理投票 |

**长期通胀控制：**
- 挖矿池有限（4亿LAC），不会无限增发
- 半衰期衰减确保释放量逐月递减
- 第3年后月释放量预计低于燃烧量，进入净通缩

### 5.3 鲸鱼操控防护

| 风险 | 对策 |
|------|------|
| 大户抛压砸盘 | 单地址24h卖出超过流通量0.5%触发冷却(+2%滑点) |
| 囤积治理权 | 投票权按根号持仓量计算: 投票权=√(持仓量) |
| 挖矿垄断 | 单地址每日挖矿奖励硬上限500LAC |
| LP操控 | 集中度监控，单LP>30%池子触发预警 |

**反鲸公式（卖出滑点附加）：**
```
附加滑点 = max(0, (卖出量/流通量 - 0.003) × 100)%
```
即卖出超过流通量0.3%的部分，每超0.01%加1%滑点，大额抛售成本极高。

### 5.4 其他风险

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 监管风险 | 中 | 高 | 去中心化治理，无预挖指控（团队锁仓12月） |
| 智能合约漏洞 | 低 | 极高 | 多重审计(Certik+OtterSec)，保险基金 |
| AI叙事退潮 | 中 | 中 | 产品实用性不依赖叙事热度 |
| Solana宕机 | 低 | 中 | 短期容忍，长期考虑跨链 |

## 七、第一年预测模型

**假设：** TGE后3个月达10,000 DAU，12个月达50,000 DAU
