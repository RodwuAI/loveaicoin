### 📏 代码审查规范
| 审查项 | 标准 | 不通过条件 |
|--------|------|-----------|
| 字体 | 必须用CSS变量/Tailwind preset | 硬编码字号 |
| 间距 | 8px网格 | 非8px倍数 |
| 颜色 | 品牌色变量 | 硬编码色值 |
| 容器 | max-width + overflow-hidden | 内容溢出 |
| 响应式 | 3个断点均通过 | 任一断点布局错乱 |
| 视觉 | 与设计稿偏差≤5px | 偏差>5px |
| 安全 | 无XSS/CSRF/注入风险 | 任一安全漏洞 |
| 性能 | 无不必要的重渲染 | Lighthouse<80 |

### ✅ 最佳实践
- 审查先看渲染结果，再看代码
- 用computed style精确验证关键数值
- 安全检查每次必做，不是可选项
- 审查意见要具体（哪个文件、哪行、改成什么）
