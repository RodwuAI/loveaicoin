# 官网验收清单 — 逐区域 Phantom 对比

## 验收原则
- **不看蜂工自评，只看渲染结果**
- **每个区域必须截图，与 Phantom 同区域截图并排对比**
- **任何一项不通过，整体不通过**
- **吴总看到之前，我必须已经发现并修完所有问题**

## 验收流程（每次修改后必须执行）

### Step 1: 截 Phantom 参照（每个区域）
1. Header（logo大小、导航、右侧按钮）
2. Hero（标题、副标题、CTA按钮、留白）
3. Feature sections（左文右卡、卡片大小、间距）
4. 过渡sections（心愿集结、下载App、尾部CTA）
5. Footer 邮件订阅区（标题、输入框、按钮）
6. Footer 导航列（列数、间距、字号）
7. Footer 底栏（版权、链接、有无logo）

### Step 2: 截我们的站（同区域）
- 同上7个区域逐一截图

### Step 3: 逐区域对比检查清单

#### Header
- [ ] Logo 大小与 Phantom 一致（不是缩略图）
- [ ] 导航居中，pill形容器
- [ ] 右侧有CTA按钮（下载App）+ 汉堡图标
- [ ] 高度、间距一致

#### Hero
- [ ] 顶部留白 ≥ 150px
- [ ] H1 居中，56px bold
- [ ] 副标题 18px light，居中
- [ ] CTA 按钮圆角、颜色正确

#### Feature Sections
- [ ] 左侧标题区：H2 30px normal（不加粗！）
- [ ] 左侧描述 + "了解更多"按钮
- [ ] 右侧卡片：有icon，不是空色块
- [ ] 卡片不溢出、不裁切
- [ ] section 间距 100px

#### 过渡 Sections
- [ ] 心愿集结：大标题居中
- [ ] 下载App：App Store + Google Play 按钮
- [ ] 尾部CTA：大标题 + 按钮

#### Footer 邮件订阅
- [ ] 标题大字 ~40px light
- [ ] 输入框 + 订阅按钮同行
- [ ] 不溢出容器

#### Footer 导航
- [ ] 4列均匀分布
- [ ] 列标题灰色小字，链接藏蓝
- [ ] 行间距舒适

#### Footer 底栏
- [ ] 只有文字版权，**没有logo**
- [ ] 右侧隐私政策、服务条款
- [ ] 分割线细

### Step 4: computed style 精确验证
对关键元素跑 JS 取 computed style，逐项对照规范值。

### Step 5: 响应式检查
- [ ] 1440px 宽度
- [ ] 1200px 宽度
- [ ] 768px 宽度（平板）
- [ ] 375px 宽度（手机）

## 红线
- 任何视觉差异 > 5px 的偏差 = 不通过
- 任何 Phantom 有我们没有的元素 = 不通过
- 任何 Phantom 没有我们有的元素 = 不通过（如底部多余logo）
