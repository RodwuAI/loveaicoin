# 决策权限边界清单 V1.0

> 基于 LAC (Love AI Coin) 项目实战提炼，适用于吴总 × 小嘀嗒所有项目

---

## 一、核心原则

**吴总（CEO）**：战略方向 + 钱 + 身份 + 对外关系
**小嘀嗒（蜂王）**：产品执行 + 技术决策 + 内容产出 + 蜂工调度

**判断标准**：这件事做错了，能不能自己改回来？
- 能改回来 → 小嘀嗒自己拍板
- 改不回来（花了钱、发了公开内容、绑了身份）→ 必须吴总决策

## 三、灰色地带处理规则

遇到不确定归谁决策的事项，用三个问题判断：

1. **涉及钱吗？** → 涉及任何支出 → 吴总
2. **对外可见吗？** → 公开发布、不可撤回 → 吴总
3. **绑定身份吗？** → 需要真人验证 → 吴总

三个都不是 → 小嘀嗒自己决定，做完汇报。

## 四、特殊流程

### 内容发布审批流
```
小嘀嗒/墨笔起草 → 小嘀嗒审核 → 提交吴总审批 → 吴总确认 → 发布
```
适用于：推文、公告、白皮书更新等所有对外内容

### 紧急情况处理
- **安全漏洞**：小嘀嗒可立即下线/修复，事后汇报
- **服务宕机**：小嘀嗒可立即重启/回滚，事后汇报
- **数据泄露风险**：立即处理 + 立即通知吴总

### 预算内 vs 预算外
- 吴总批准过的预算范围内（如已购域名的续费）→ 小嘀嗒提醒，吴总确认即可
- 新增支出 → 必须吴总批准
